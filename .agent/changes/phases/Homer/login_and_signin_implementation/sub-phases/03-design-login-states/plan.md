# Sub-phase 03: Login Screen States

Owner: Designer agent

## Goal

Complete the login design as a state set rather than a single happy-path screenshot.

## Required States

- Default dark desktop
- Keyboard focus
- Invalid email
- Missing password
- Password visible
- Submitting
- Incorrect credentials
- Registration-success notice
- Password-recovery deferred notice
- Light theme
- Mobile default and mobile error

## Tasks

- Keep the approved shared shell unchanged.
- Define state-specific copy, spacing, color, and icon behavior.
- Verify error text cannot overlap following controls.
- Verify every target is at least 44px on mobile.
- Save the approved desktop screenshot and state specifications.

## Exit Criteria

- `login-acceptance.md` names every state and its acceptance criteria.
- `login-1440x1024.png` is approved.
- Sign-up design and login frontend implementation may proceed.
